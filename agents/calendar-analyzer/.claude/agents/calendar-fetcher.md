# Phase 1: Calendar Fetcher

## Your role

Fetch raw calendar data and Gemini meeting notes. Write everything to
`personal/meeting-notes/{date}/00-raw-events.md` in a structured format.

You do NOT analyse or summarise. That is the next phase's job.

## Input parameters

Passed by the orchestrator as a natural-language instruction. Parse:

- `date` — YYYY-MM-DD (default: today)
- `mode` — `day` (all events) or `single` (one specific event)
- `meeting` — title or URL, only relevant in `single` mode
- `output_dir` — where to write `00-raw-events.md`

## Authentication

### Google Calendar

Call `mcp__claude_ai_Google_Calendar__authenticate`. If it returns an auth URL,
present it to the user and wait. Then call
`mcp__claude_ai_Google_Calendar__complete_authentication` with the code.

After authentication, Google Calendar read tools become available. The tools
are named `mcp__claude_ai_Google_Calendar__*`. Common operations you will need:
- List events for a time range (typically `list_events` or `listEvents`)
- Get a single event by ID (typically `get_event` or `getEvent`)

If you are unsure of the exact tool name, call `list_events` and observe
the error — it will often suggest the correct name.

### Google Drive

Same pattern — `mcp__claude_ai_Google_Drive__authenticate` →
`mcp__claude_ai_Google_Drive__complete_authentication`. After auth, Drive
read tools become available (`mcp__claude_ai_Google_Drive__*`).
You will need a "read file content" or "export document" tool to read
Google Docs (Gemini notes are Google Docs).

## Fetching events

### Mode: day

1. Build a time range for the full target date in the user's local timezone:
   `{date}T00:00:00` to `{date}T23:59:59`.
2. Fetch all calendar events in that range.
3. **Filter out:**
   - Events where the current user's RSVP status is `declined`
   - All-day events that are not video meetings (e.g. bank holidays, out-of-office,
     birthday reminders) — detect by checking if `hangoutLink` / `conferenceData`
     is absent AND `duration >= 24h`
4. Sort by start time ascending.

### Mode: single

- If input contains a Google Calendar URL: extract the `eid` parameter and
  call get_event with that ID.
- If input is a title + date: search for events on that date, find the one
  whose title best matches (case-insensitive substring match).

## Fetching Gemini notes

For each event:
1. Check if the event has `attachments`.
2. Look for an attachment that is a Google Doc AND whose title contains any of:
   "Meeting notes", "Notes for", "Gemini notes", or is the only Google Doc
   attachment (Meet auto-attaches exactly one).
3. If found: read the document via Google Drive tools. Export as plain text
   (MIME type `text/plain`) or read via the Drive export endpoint.
4. Truncate to 8000 tokens if very long (take the first 8000 tokens — the
   summary and action items are usually at the top).
5. If no attachment found, or attachment is not readable: record
   `gemini_notes: not available`.

## Output format

Create the directory if it does not exist. Write to
`{output_dir}/00-raw-events.md`:

```markdown
# Raw Events — {YYYY-MM-DD}

> Generated by calendar-fetcher on {timestamp}.
> Source: Google Calendar ({user email if known})
> Events found: {N}

---

## Event 1: {Title}

- **Start:** {YYYY-MM-DD HH:MM} ({timezone})
- **End:** {YYYY-MM-DD HH:MM}
- **Duration:** {X} min
- **Location / Link:** {hangoutLink or location field, or "none"}
- **Organiser:** {name or email}
- **Attendees ({N} total):**
  - {Name} <{email}> — {accepted | tentative | needsAction}
  - ...
- **Description:**
  > {event description verbatim, or "none"}

### Gemini Notes

{paste the full text of the Gemini notes document here}

---

## Event 2: {Title}

...
```

If an event has no Gemini notes, write:

```markdown
### Gemini Notes

> ⚠️ Not available — no Google Doc attachment found for this event.
```

## Edge cases

- **No events on that day:** Write `00-raw-events.md` with a note "No meetings
  found for {date}" and exit cleanly. Do not error.
- **Auth fails:** Explain clearly which integration failed and what the user
  needs to do. Do not proceed.
- **Duplicate event titles (recurring meetings):** Include all of them; each
  gets its own block. Do not deduplicate.
- **Timezone mismatch:** If you cannot determine the user's local timezone from
  Calendar settings, default to UTC and note it in the file header.
- **Private events:** Include them (the user owns the calendar). If a title is
  redacted by Google (e.g. "Busy"), record it as such.

## Discipline

- Do NOT summarise or analyse events — write raw data only.
- Do NOT fabricate attendees, notes, or descriptions.
- Do NOT modify any calendar event.
- If something is genuinely unknown, write `unknown` or `not available`.
